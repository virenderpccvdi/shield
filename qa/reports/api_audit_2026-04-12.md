## Bugs Found

### BUG-1: Rewards Leaderboard returns 500 (HIGH)
- **Endpoint:** `GET /api/v1/rewards/leaderboard`
- **Error:** `MissingServletRequestParameterException: Required request parameter 'tenantId' for method parameter type UUID is not present`
- **Impact:** Leaderboard crashes for both admin and customer when tenantId is not provided as query param
- **Expected:** For CUSTOMER role, tenantId should be extracted from JWT claim. For GLOBAL_ADMIN, it should be optional or return cross-tenant results.
- **Workaround:** Works when `?tenantId=<uuid>` is appended manually

### BUG-2: Customer cannot access own child's DNS rules (HIGH)
- **Endpoint:** `GET /api/v1/dns/rules/{profileId}`
- **Error:** `403 FORBIDDEN - "Cannot verify profile ownership"`
- **Impact:** CUSTOMER role users cannot view DNS filtering rules for their own children. This blocks a core feature.
- **Note:** Other DNS endpoints (schedules, time-limits, access-schedules, budgets) work fine for the same profileId with the same token. Only the rules endpoint is broken.

### BUG-3: Schedule grid has duplicate day keys (MEDIUM)
- **Endpoint:** `GET /api/v1/dns/schedules/{profileId}`
- **Response contains:** Both `"Mon"` (old format) and `"monday"` (new format) keys with different values
- **Impact:** Clients may parse inconsistently. Only 2 keys returned instead of 7 days, suggesting only Monday has schedule data in the new format. Old data (`"Mon"`) was not migrated.

### BUG-4: Time limits shows exhausted=true when no budget set (LOW)
- **Endpoint:** `GET /api/v1/dns/time-limits/{profileId}`
- **Response:** `dailyBudgetMinutes: null, usedMinutes: 0, remainingMinutes: -1, exhausted: true`
- **Impact:** When no daily budget is configured (null), the system reports `exhausted: true` and `remainingMinutes: -1`. Should report `exhausted: false` with unlimited remaining when budget is null.

### BUG-5: Admin service unhealthy / Swagger 500 (HIGH)
- **Endpoint:** `/docs/admin/swagger-ui/index.html` (via gateway) and direct port 8290
- **Impact:** Admin service on port 8290 is running (listening) but not responding to HTTP requests (curl exit code 56 = recv failure). HikariCP pool logs show repeated "Failed to validate connection... This connection has been closed" warnings.
- **Root cause:** HikariCP maxLifetime likely exceeds PostgreSQL/HAProxy connection timeout, causing stale connections.

### BUG-6: Public HTTPS API returns 404 (CRITICAL)
- **Endpoint:** `POST https://shield.rstglobal.in/api/v1/auth/login` (any API path)
- **Impact:** All API calls from the public internet return 404 (website 404 page). DNS resolves to Azure LB IP 135.235.191.247, which serves the static website but does not proxy API paths to the gateway.
- **Note:** Local nginx HTTPS works correctly (`curl -k https://localhost/api/v1/auth/login -H 'Host: shield.rstglobal.in'` returns 200). The issue is that DNS no longer points to this server.

### BUG-7: Rewards tasks returns different results for admin vs customer (LOW)
- **Endpoint:** `GET /api/v1/rewards/tasks`
- **Admin response:** `[]` (empty)
- **Customer response:** Tasks list with data
- **Note:** This may be by design (admin has no tenant context for task filtering), but the behavior is undocumented and could confuse API consumers.

### BUG-8: Admin sees empty children list (LOW)
- **Endpoint:** `GET /api/v1/profiles/children`
- **Admin response:** `[]` (empty) despite children existing in the system
- **Note:** GLOBAL_ADMIN has no customerId, so the profile service returns empty. May be by design but admin should arguably see all children across tenants.

## Recommendations

1. **CRITICAL - Fix public API routing (BUG-6):** Either update Azure LB / AKS ingress to proxy `/api/` paths to the gateway, or update DNS to point back to the origin server. This completely blocks mobile app and external integrations.

2. **HIGH - Fix customer DNS rules access (BUG-2):** The ownership verification in `DnsRulesController` is not matching the customer's userId to their child profiles. Other DNS controllers (schedules, time-limits) handle this correctly -- align the rules controller.

3. **HIGH - Fix leaderboard tenantId (BUG-1):** Extract tenantId from JWT `tenant_id` claim for CUSTOMER role instead of requiring a query parameter. Make the param optional for GLOBAL_ADMIN.

4. **HIGH - Fix admin service health (BUG-5):** Set `spring.datasource.hikari.max-lifetime` to a value below PostgreSQL/HAProxy idle timeout (e.g., 580000ms for a 600s HAProxy timeout). Also add `spring.datasource.hikari.connection-test-query=SELECT 1`.

5. **MEDIUM - Clean up schedule grid (BUG-3):** Run a data migration to convert old short-name keys (Mon, Tue...) to full lowercase (monday, tuesday...) and remove duplicates.

6. **LOW - Fix time-limits logic (BUG-4):** When `dailyBudgetMinutes` is null, return `exhausted: false` and `remainingMinutes: null` (or -1 meaning unlimited) rather than marking as exhausted.
