Why Shield?

Protection That Actually Works — Everywhere

Most parental controls only work on the home WiFi. The moment your child leaves the house or switches to 4G data, the protection disappears.

Shield uses DNS-over-HTTPS — a technology that routes all DNS lookups through your child's personal filtered server. It works at the operating system level, before any app can make a network connection, on any network.

What's New in v2.1.5

Child Tasks — Status Tracking Now Correct v2.1.5

Fixed a critical bug where all child tasks appeared as "To Do" forever — the app was checking a non-existent isCompleted boolean instead of the API's status field (PENDING → SUBMITTED → APPROVED). Tasks now show four states with proper icons: To Do, Waiting for Approval (hourglass), Completed (green tick), and Rejected (re-doable). The parent Rewards screen and Approval Requests screen now correctly show tasks awaiting review.

Type Safety — No More Runtime CastError Crashes v2.1.5

Fixed 16 screens using unsafe raw.cast<Map>() — replaced with safe whereType<Map>().toList() that skips bad elements instead of crashing. Fixed 10+ as int? casts on API-returned numbers (JSON can return 2.0 as a double, causing type 'double' is not a subtype of 'int' crashes) — now uses (as num?)?.toInt() throughout: points, threshold, screen time, duration, customer count, and domain visit count.

Settings & Data — Clear History, PIN Validation Fixed v2.1.5

Clear Browsing History now actually calls the API and deletes data for all children's profiles (was a no-op before). Parent PIN now validates digits-only with specific error messages. Privacy Policy, Terms of Service, and Help links now open in the browser. Fixed a crash in the token refresh retry path where the response envelope was not unwrapped for the retried request.

Critical Fix — Child Device DNS Blocking All Websites v2.1.4

Fixed a critical bug where the Shield VPN on child devices blocked ALL websites including allowed ones like Google. Root cause: the VPN's DoH forwarding used InetSocketAddress(hostname) which triggered OS DNS resolution — routed back through the VPN's own fake DNS — causing an infinite recursive loop. All DNS queries timed out. Fix: the DoH server IP is now resolved once before the VPN tunnel starts, so socket connections use the IP directly (hostname kept for TLS SNI). Also cleared a stale budget-exhausted flag on the Disha test profile.

Login Overflow, Logout & Schedule Screen Fixed v2.1.3

Fixed 21px overflow on the login screen (home indicator inset was ignored). Fixed logout not working — GoRouter redirect alone was unreliable; explicit navigation added. Fixed Internet Schedule screen showing wrong data — backend uses a 24-slot hourly grid format (1=blocked, 0=allowed) which is now correctly converted to the per-day start/end time UI.

Parent Controls — DNS, Filters, Schedule, Bedtime All Fixed Fixed

Fixed critical bug where all child profile screens showed empty data. API response was not unwrapped. Fixed: Safe Filters saves categories correctly; Time Limits uses dailyBudgetMinutes; Bedtime uses bedtimeStart/bedtimeEnd; Homework Mode uses POST start/stop endpoints.

API Endpoint Fixes — DNS, Rewards & Notifications Fixed

Fixed 12 mismatched API paths that caused "Failed to Load" on DNS schedule, homework mode, bedtime, safe filters, rewards tasks, points bank, achievements, daily check-in, notifications, browsing history, and app usage screens. Added daily check-in endpoint (5 bonus points). All backend services restarted with improved connection pool settings.

Child Device Setup — No Re-Login Required Fixed

Parents who are already logged in no longer see an email/password prompt when linking a child device. The app detects the active session, loads child profiles automatically, and jumps straight to the "Select Child" step. The password screen now only appears on a fresh (unauthenticated) device.

Installation Guide — Shield uses parental control permissions that require a one-time setup
1
Allow installs from your browser

Open Settings → Apps → Special App Access → Install Unknown Apps, find your browser (Chrome / Files), and tap Allow from this source.

2
Download & install

Tap the Download button above, open the downloaded file, and tap Install. No need to disable Play Protect — v2.0.0 uses standard Android APIs.

3
Grant Usage Access (child device only)

For app blocking to work on the child's device, go to Settings → Apps → Special App Access → Usage Access and enable Shield.

Sign in & set up

Open Shield, log in as parent or scan the QR code on the child's device to link it to your family account.
